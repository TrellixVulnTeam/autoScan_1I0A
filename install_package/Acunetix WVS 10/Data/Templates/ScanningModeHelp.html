Scanning Mode
On some web applications some parameters are used as an "action" parameter. The value of these "action" parameters will determine how the other parameters will be interpreted. The scanning mode determines how parameter values are tested during scanning.

Quick Only the first value from every parameter will be tested.

Heuristic (default) WVS will try to automatically determine which parameters require complex testing.

Extensive

All possible combinations for every parameter will be tested.
When there are a lot of parameters/combinations, this mode will generate a lot of HTTP requests.
Scan Speed
Scan Depth

These modes should be selected according to the target application. For example, shopping cart product_id values 1,2,3,4 would not need to be all tested since they are not used as action parameters. In this case, ‘Quick’ scanning mode would be the correct setting. When using the ‘Extensive’ scanning mode a large number of requests will be generated. Therefore when scanning large applications, it is recommended to segment the scan, and target different areas of the site separately.
