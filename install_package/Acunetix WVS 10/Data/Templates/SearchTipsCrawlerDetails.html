Search Tips for Crawler Tool (custom search mode)
Notes:
  • The search is case insensitive.
  • You can use the boolean operator OR(|) to perform searches like acunetix|wvs
    (it will match pages containing either acunetix or wvs).
  • To exclude a word from the search you can use the minus sign (-). E.g acunetix -wvs
    (it will match pages containing acunetix but not wvs).
  • You can search for a certain expression by including it within quotes. Like "index of /secure"
  • Global modifiers should be placed at the start of the search query. They will be applied for all the search terms.
  • You can replace one or more words using the * operator. E.g. "web * scanner"
    (it will match web vulnerability scanner).

Search modifiers :
Search Modifier Description
inurl: Use this modifier if you want to restrict your search to the URL part of the page.
allinurl: is the related global modifier.
intext: Use this modifier to restrict your search within the text of the page.
allintext: is the related global modifier.
intitle: Use this modifier to restrict your search to the title of the page.
allintitle: is the related global modifier.
inheaders: Restict your search to the HTTP headers of the page.
allinheaders: is the related global modifier.
filetype:, ext: Search for files of a specific type.
link: Search for links to a certain page.

Special searches :
Special search Description
?haveinputs Returns all the files that have inputs (GET or POST).

Examples:
  • inurl:CVS will match files with CVS within their URL (such as /CVS/)
  • allinurl:CVS Repository will match files containg CVS and repository within their URL (such as /CVS/Repository)
  • ext:php will return all the PHP scripts from the website.
  • intitle:"index of" will return directory listings
